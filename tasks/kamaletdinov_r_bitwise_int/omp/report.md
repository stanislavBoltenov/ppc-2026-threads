## 1. Контекст

Версия OpenMP использует многопоточность с общей памятью для ускорения этапа сортировки подсчетом в рамках каждого разряда.

## 2. Постановка задачи

Перенос последовательной логики на рельсы OpenMP с сохранением устойчивости сортировки. Реализация основана на
последовательной версии алгоритма.

## 3. Базовый алгоритм

В основе лежит поразрядная сортировка, где параллелизм внедряется в критические по времени участки — подсчет частот цифр
и перераспределение элементов.

В отличие от последовательной версии, где гистограмма частот строится за один проход по всему массиву, в OMP-версии:

1. **Параллельный подсчет**: Входной массив делится на блоки. Каждый поток обрабатывает свой блок и заполняет локальную
   гистограмму частот разрядов. Это позволяет избежать конфликтов при записи в общую память на этапе накопления
   статистики.
2. **Слияние и смещения**: После завершения подсчета локальные гистограммы суммируются в глобальную. На основе
   глобальной гистограммы вычисляются префиксные суммы (стартовые позиции для каждой цифры). В последовательном
   алгоритме этот шаг выполняется сразу после одного прохода.
3. **Параллельная перестановка**: Каждый поток, используя глобальные стартовые позиции и свои локальные данные о
   количестве цифр, вычисляет уникальные индексы для записи элементов в результирующий массив. Это ключевое отличие: в
   последовательной версии индекс инкрементируется линейно, а здесь он вычисляется на лету для каждого потока, что
   исключает гонки данных при записи.

## 4. Схема распараллеливания

- **Область параллелизма**: Цикл прохода по массиву для подсчета гистограммы и цикл перестановки элементов.
- **Данные**:
  - `shared`: входной массив, глобальный массив счетчиков.
  - `private`: итераторы, локальные значения.
- **Синхронизация**: Каждый поток заполняет локальную гистограмму частот (массив из 10 элементов). Затем происходит
  слияние локальных гистограмм в глобальную.
- **Планирование**: Используется `schedule(static)` для равномерного распределения блоков массива между потоками.

## 5. Детали реализации

Файлы: `omp/include/ops_omp.hpp`, `omp/src/ops_omp.cpp`
Используется директива `#pragma omp parallel` для создания пула потоков. Критические секции отсутствуют, так как каждый
поток пишет в свой локальный буфер или в строго определенную позицию результирующего массива на основе заранее
рассчитанных смещений.

Пример ключевого фрагмента:

```cpp
#pragma omp parallel
{
    int thread_id = omp_get_thread_num();
    // каждый поток считает гистограмму для своей части данных
    for (int i = start; i < end; ++i) {
        local_counts[thread_id][get_digit(data[i])]++;
    }
}
```

## 9. Выводы

OpenMP версия показала заметное ускорение на 3 потоках (1.64x). Однако при дальнейшем увеличении числа потоков до 6 и 9
ускорение практически не растет, а эффективность падает. Это связано с тем, что накладные расходы на управление потоками
и синхронизацию (слияние гистограмм) начинают доминировать над вычислительной частью на данном объеме данных.

## 10. Код реализации

```cpp
void CountingSortByDigit(std::vector<int> &arr, int exp) {
  const int n = static_cast<int>(arr.size());
  const int thread_count = omp_get_max_threads();
  std::vector<std::array<int, 10>> local_counts(thread_count);

#pragma omp parallel default(none) shared(arr, exp, local_counts, n) num_threads(thread_count)
  {
    const int tid = omp_get_thread_num();
    auto &current = local_counts.at(tid);
    current.fill(0);

#pragma omp for schedule(static)
    for (int i = 0; i < n; i++) {
      const int digit = (arr.at(i) / exp) % 10;
      current.at(digit)++;
    }
  }

  // Сборка глобальной гистограммы и расчет смещений...
  // (с использованием thread_offsets для бесконфликтной записи)

#pragma omp parallel default(none) shared(arr, exp, output, n, thread_count, thread_offsets) num_threads(thread_count)
  {
    const int tid = omp_get_thread_num();
    auto positions = thread_offsets.at(tid);

#pragma omp for schedule(static)
    for (int i = 0; i < n; i++) {
      const int digit = (arr.at(i) / exp) % 10;
      output.at(positions.at(digit)++) = arr.at(i);
    }
  }
}
```
