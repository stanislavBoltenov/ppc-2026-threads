## 1. Контекст

Данная версия использует стандартную библиотеку потоков `std::thread` для ручного управления параллелизмом. Это
требует явного разделения данных и синхронизации.

## 2. Постановка задачи

Реализация параллельной поразрядной сортировки с использованием низкоуровневых примитивов `std::thread`. Реализация
основана на последовательной версии алгоритма.

## 3. Базовый алгоритм

В STL-реализации параллелизм управляется вручную через пул потоков или их создание на каждой итерации. Основные
отличия от последовательной версии заключаются в способе обработки данных.

Если в последовательном алгоритме весь массив обрабатывается одним потоком, то здесь:

1. **Статическое разбиение**: Размер массива делится на количество доступных потоков. Каждый поток получает начальный
   и конечный индекс своего блока, что требует дополнительной логики по расчету границ, отсутствующей в SEQ.
2. **Локальный подсчет**: Потоки запускаются параллельно, каждый считает гистограмму цифр в своем диапазоне. Результаты
   сохраняются в двумерный массив (или вектор векторов). В отличие от SEQ, где есть одна гистограмма, здесь их
   количество равно числу потоков.
3. **Барьерная синхронизация**: Основной поток дожидается завершения всех рабочих потоков через `join()`. После этого он
   последовательно суммирует локальные гистограммы и вычисляет глобальные смещения. В последовательной версии этот
   этап отсутствует, так как данные не разделяются.
4. **Перераспределение**: Потоки запускаются повторно (или продолжают работу) для перемещения элементов в выходной
   буфер. В отличие от SEQ, каждый поток знает свое локальное смещение относительно глобального начала блока каждой
   цифры, что гарантирует бесконфликтную запись.

## 4. Схема распараллеливания

- **Разбиение**: Входной массив делится на `n / num_threads` блоков.
- **Потоки**: Создается массив объектов `std::thread`. Каждый поток выполняет вычисления для своего блока.
- **Локальные данные**: Каждый поток имеет свой локальный массив счетчиков для исключения гонок данных.
- **Синхронизация**: Использование `thread.join()` для ожидания завершения всех потоков перед переходом к следующему
  разряду или финальной сборке.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`
Ручное управление требует аккуратности в вычислении индексов границ блоков.

Пример структуры:

```cpp
std::vector<std::thread> threads;
for (int i = 0; i < num_threads; ++i) {
    threads.emplace_back([&, i]() {
        // вычисление границ для потока i и дальнейшая обработка данных
    });
}
for (auto& t : threads) t.join();
```
