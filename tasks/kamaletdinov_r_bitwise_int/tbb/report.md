## 1. Контекст

Версия с использованием Intel oneTBB применяет парадигму задач (task-based parallelism) и алгоритм Work-stealing для
более гибкого распределения нагрузки.

## 2. Постановка задачи

Использование примитивов TBB для распараллеливания этапа подсчета и слияния гистограмм. Реализация основана на
последовательной версии.

## 3. Базовый алгоритм

Алгоритм использует `tbb::parallel_reduce` для эффективного вычисления гистограмм распределения цифр. В отличие от
последовательной реализации, где массив сканируется один раз одним потоком, в TBB-версии процесс декомпозирован:

- **Разделение (Split)**: TBB рекурсивно делит входной диапазон массива на поддиапазоны до достижения `grainsize`. Это
  динамический процесс, в SEQ-версии данные обрабатываются линейно без разделения.
- **Подсчет (Map)**: В каждом поддиапазоне поток вычисляет локальную гистограмму для текущего разряда. Таким образом,
  вместо одной глобальной гистограммы в SEQ, здесь создается множество временных объектов гистограмм.
- **Слияние (Reduce)**: Метод `join` в TBB объединяет две гистограммы (поэлементное сложение массивов счетчиков), что
  позволяет безопасно агрегировать данные из разных потоков без использования явных мьютексов. В последовательном
  алгоритме этот шаг отсутствует, так как результат накапливается сразу в итоговый массив счетчиков.
- **Префиксные суммы**: После завершения редукции рассчитываются позиции записи, и массив перераспределяется. Это
  единственный этап, который логически совпадает с SEQ, но выполняется над агрегированными данными из всех потоков.

## 4. Схема распараллеливания

- **Примитив**: `tbb::parallel_for`.
- **Логика**: Массив делится на части. Для каждой части вычисляется локальная гистограмма.
- **Grainsize**: Используется значение по умолчанию или автоматически подбираемое TBB runtime.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`
Основное преимущество TBB в данной задаче — автоматическое управление балансировкой нагрузки.

Пример использования:

```cpp
tbb::parallel_reduce(
    tbb::blocked_range<size_t>(0, n),
    LocalHist{},
    [](const tbb::blocked_range<size_t>& r, LocalHist init) -> LocalHist {
        for (size_t i = r.begin(); i < r.end(); ++i) {
            init.counts[get_digit(data[i])]++;
        }
        return init;
    },
    [](LocalHist a, const LocalHist& b) -> LocalHist {
        for (int i = 0; i < 10; ++i) a.counts[i] += b.counts[i];
        return a;
    }
);
```

void CountingSortByDigit(std::vector<int> &arr, int exp) {
  const int n = static_cast<int>(arr.size());
  const int num_parts = tbb::this_task_arena::max_concurrency();
  std::vector<std::array<int, 10>> part_counts(num_parts);

  tbb::parallel_for(0, num_parts, [&](int part) {
    part_counts.at(part).fill(0);
    const int begin = part * n / num_parts;
    const int end = (part + 1) * n / num_parts;
    for (int i = begin; i < end; i++) {
      const int digit = (arr.at(i) / exp) % 10;
      part_counts.at(part).at(digit)++;
    }
  });
