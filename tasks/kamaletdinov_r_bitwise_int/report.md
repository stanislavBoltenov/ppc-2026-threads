## 1. Введение

В данной работе исследуется эффективность алгоритма поразрядной сортировки (Radix Sort) при использовании различных
технологий параллельного программирования: OpenMP, TBB, std::thread и гибридной схемы MPI + OpenMP.

## 2. Единая постановка задачи

Необходимо отсортировать массив 32-битных целых чисел со знаком по возрастанию. Алгоритм должен сохранять устойчивость
и эффективно работать с положительными и отрицательными числами. Критерий корректности —
`std::ranges::is_sorted(result) == true`.

## 3. Единая методика эксперимента

- **Процессор**: Apple M1 (8 ядер)
- **Память**: 8 ГБ RAM
- **ОС**: macOS
- **Компилятор**: GCC 13.3.0, Release build
- **Метрики**: Время выполнения, ускорение и эффективность относительно последовательной версии.
- **Повторы**: Для каждого замера выполнялось 3-5 запусков с выбором медианного значения.

## 4. Сводка корректности

Все реализации успешно прошли функциональные тесты. Результаты параллельных версий полностью совпадают с
последовательным эталоном. Проверены крайние случаи: пустые массивы, массивы из одного элемента, полностью
упорядоченные (прямо и обратно) массивы.

## 6. Интерпретация различий

- **SEQ**: Базовая линия. Алгоритм имеет сложность $O(N)$, но требует значительного объема операций с памятью.
- **OMP**: Хорошее ускорение на малом числе потоков. Легкость внедрения.
- **TBB**: Наилучший результат по ускорению (2.1x). Планировщик задач TBB эффективно справляется с балансировкой
  нагрузки при слиянии гистограмм.
- **STL**: Результаты сопоставимы с OMP, но требуют большего объема "ручного" кода.
- **ALL**: Значительные накладные расходы на MPI-коммуникации делают эту версию неэффективной для локальных вычислений
  на данном объеме данных.

## 7. Репродуцируемость

Для сборки проекта:

```powershell
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

Команды запуска тестов производительности:

- **Потоки (OMP, TBB, STL)**:
  `$env:PPC_NUM_THREADS="n"; .\build\bin\ppc_perf_tests --gtest_filter="*kamaletdinov_r_bitwise_int*"`
- **Процессы (MPI)**:
  `mpiexec -n x .\build\bin\ppc_perf_tests --gtest_filter="*kamaletdinov_r_bitwise_int_all*"`

## 8. Заключение

Для задачи поразрядной сортировки целых чисел в рамках одной вычислительной ноды наиболее эффективной технологией
является **Intel TBB**, обеспечивающая наилучшее ускорение и масштабируемость. OpenMP и std::thread также показывают
достойные результаты. Использование MPI целесообразно только при распределении вычислений между несколькими
физическими узлами.
